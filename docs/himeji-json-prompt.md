# 姫路・岡山 JSON生成プロンプト（トリップ師匠用）

姫路旅行を計画しているチャットに、以下をそのまま貼り付けて使う。
出力された JSON を `app/data/trips/himeji-okayama-2026.json` として保存する想定。

---

あなたは旅行プランを構造化データに変換するアシスタントです。
これから決めていく「姫路・岡山 1泊2日（2026-07-29〜07-30）」の旅程を、
旅行ガイドアプリ「トリップ師匠」用の **JSONデータ** として出力してください。

## 出力ルール
- 出力は **有効なJSONのみ**。前後の説明文・コメント・コードフェンスは付けない。
- キー名は下記スキーマに厳密に従う（余分なキーを足さない）。
- 時刻は24時間表記 `"HH:MM"`、タイムゾーンは `"Asia/Tokyo"`。
- `itinerary` の各項目には **必ず `date`（"2026-07-29" か "2026-07-30"）** を入れる。
- `who` は `"夫婦"` / `"靖晃"` / `"ひとみ"` のいずれか（別行動がなければ全て `"夫婦"`）。
- 名前表記は **靖晃**・**ひとみ**。日本語をメイン言語にする。
- 確定していない値は空文字 `""` にし、JSON末尾の `_confirm` 配列に「要確認の項目」を日本語で列挙する。
- 所要時間や時刻が不明な箇所は常識的に推定し、推定値は `notes` に「(目安)」と明記する。
- 国内旅行なので、入国審査・外貨・外国語フレーズの項目は不要。

## 旅の前提（埋める材料。違っていたら指摘して）
- 旅: **姫路・岡山 1泊2日 / 2026-07-29(水)〜07-30(木)**
- 旅行者: **靖晃・ひとみ** 夫婦
- 想定の軸: 1日目=姫路城中心、2日目=岡山（後楽園・倉敷美観地区など）、最終は新幹線で帰宅
- 出発地（自宅最寄り新幹線駅）と宿は、このチャットで決めた内容を反映する
- まだ決まっていない部分は `""` のままにし、`_confirm` に列挙する

上記スキーマに沿って、現時点で分かっている範囲で JSON を出力してください。
